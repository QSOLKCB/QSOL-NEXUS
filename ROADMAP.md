# QSOL NEXUS Roadmap

The 1.0 line is deliberately frozen at three laboratories and one shared artifact system. New subjects enter only after their engine, determinism, replay, security, and claim-boundary contracts are explicit.

## v1.0.0 — Field Explorer

Release-candidate implementation:

- [x] zero-install `index.html` application shell;
- [x] fixed-order classic scripts under one versioned namespace;
- [x] Canonical Strict, Replay Safe, and explicit Creative modes;
- [x] canonical JSON, bundled SHA-256, seeded PRNG, decimal rules, scheduling and cancellation;
- [x] offline PCM16, WAV, deterministic ZIP and signal fingerprinting;
- [x] frozen recipes, observation contracts, manifests, lineage and recompute-not-trust replay;
- [x] mandatory engine claim classes and boundaries;
- [x] Qutrit Field Laboratory;
- [x] Tensor Resonance Laboratory;
- [x] Galaxy Model Laboratory with model-status hierarchy;
- [x] IndexedDB persistence with visible session fallback;
- [x] browser-native known-answer, engine, replay, tamper and security tests;
- [x] example presets, fixtures, documentation and archival metadata.

Release gate:

- [ ] execute the golden suite in current Chromium, Firefox, Edge and Safari where available;
- [ ] record Canonical Strict hash agreement for every supported browser;
- [ ] archive representative bundles from all three laboratories;
- [ ] publish the tagged source and Zenodo record.

Unchecked release gates require environment-specific validation and are not silently claimed by the source bundle.

## v1.1 — Workbench depth

- cross-laboratory handoff receipts: Qutrit covariance → Tensor, Galaxy covariance → Tensor, Galaxy residuals → Qutrit;
- richer A/B experiment comparison and observation deltas;
- canonical PNG encoder for identity-bearing plots;
- additional Qutrit custom-graph editor and user-painted field import;
- additional named MOND interpolation functions as separate model IDs;
- bounded Replay Safe residual bootstrap;
- expanded archival example corpus.

## v1.2 — Additional SPECTRAL laboratories

- photoacoustic laboratory adapter;
- photon-sphere and whispering-gallery laboratory adapter;
- E8 geometry engine after its independent claim and performance contracts are reviewed;
- optional longer PCM profiles with quota-aware persistence.

## v1.3 — Sandboxed AIMM adapter

- explanations and completed-observation summaries;
- preset suggestions and creative narration;
- music-production prompt generation;
- replay-failure explanations;
- capability-limited interface with no frozen-recipe or result mutation;
- no implicit network access and no authority to certify scientific validity.

## Deferred beyond 1.x

- LLM integration inside the trusted numerical core;
- physical quantum-hardware connections;
- full QEC decoder execution;
- WebGPU or GPU identity-bearing computation;
- live microphone or camera inputs;
- remote collaboration or hosted experiment registry;
- dynamically downloaded plugin marketplace;
- adaptive MCMC and large batch galaxy fitting;
- mobile-first redesign.

## Admission rule for a new engine

A proposed engine must provide a stable ID and semantic version, parameter and input schemas, deterministic-mode declaration, compile/render/observe/validate lifecycle, bounded execution, known-answer fixtures, complete claim boundary, one-way sonification mapping, artifact outputs, replay strategy, threat review, attribution, and an upgrade policy. A compelling animation is not an exception to this rule, even when it arrives carrying diplomatic papers.
